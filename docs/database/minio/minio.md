# MinIO

Мы используем MinIO в качестве решения для хранения медиафайлов (аватары пользователей, превьюшки фильмов, сами фильмы и т.д.).

> В теории можно рассмотреть переезд на TimeWeb Object Storage или другой облачный s3 сервис.

## Бакеты

- actors (фотографии актёров)
- avatars
- posters
- trailers
- medias (фильмы и сериалы)

## Наполнение

Чтобы избежать хранения больших файлов в репозитории, мы храним тестовые медиафайлы на гугл диске.

Чтобы заполнить локальный minio этими файлами, выполните следующие команды:

```bash
# Заполним базу медиафайлов в папку testdata/minio
# Для запуска необходим быть конфиг rclone с акцес токеном к гугл диску
make minio-pull
# Запустим minio
make minio-start
# Создадим бакеты (хардкод в makefile)
make minio-create
# Заполним minio тестовыми файлами
make minio-fill
```
