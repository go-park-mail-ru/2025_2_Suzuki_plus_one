## USER
### Таблица пользователей
```
{user_id} → username, email, password_hash, created_at, updated_at
{email} → user_id, username, password_hash, created_at, updated_at
```
### Соответствие нормальным формам
- **1НФ**: Соблюдена атомарность данных
- **2НФ**: Все неключевые атрибуты полностью функционально зависят от первичного ключа user_id
- **3НФ**: Отсутствуют транзитивные функциональные зависимости между неключевыми атрибутами
- **НФБК**: Все детерминанты функциональных зависимостей (user_id, email) являются потенциальными ключами
